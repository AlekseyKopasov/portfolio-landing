## 4. Дизайн: 8-bit / NES / CRT / 90-е (Dendy-ностальгия)

### Концепция

«Карьера как игра на приставке»: секции — уровни, стек — инвентарь, кейсы — пройденные стадии, форма — **PRESS START / INSERT COIN**.

### Палитра NES

| Роль | HEX |
|------|-----|
| Фон | `#1D1D6E` |
| Панели | `#5C94FC` |
| Текст | `#FCFCFC` |
| Акцент / CTA | `#FCE94F` |
| Успех | `#00A800` |
| Ошибка | `#E40058` |
| Обводка | `#000000` |
| Доп. | `#F8B800`, `#BCBCBC` |

### Типографика

- Заголовки: **Press Start 2P**
- Текст: **JetBrains Mono** или **VT323**, body ≥ 16px (лучше 18px на mobile)

### CRT (CSS)

- Scanlines (`repeating-linear-gradient`), лёгкий `text-shadow`;
- `image-rendering: pixelated` для пиксель-графики;
- мигание «PRESS START» — с отключением при `prefers-reduced-motion`.

### БЭМ (основные блоки)

`page`, `hero`, `level-card`, `inventory`, `case-card`, `form`, `chip`.

### Секции (уровни)

| Level | Содержание |
|-------|------------|
| TITLE SCREEN | Имя, Senior Fullstack / Frontend-heavy, ~8 лет опыта |
| PLAYER STATS | Стек-чипы |
| HOW TO PLAY | Подход к задачам + AI в работе |
| STAGES CLEARED | Кейсы (NDA-safe) |
| CONTINUE? | Форма + контакты |

### A11y (WCAG)

- Контраст жёлтого на синем — проверить;
- Видимый `:focus` у интерактива;
- `aria-live` для статусов формы;
- `prefers-reduced-motion` — без мигания и scanlines.

## 6. Форма обратной связи

### Поля

- имя (обязательно);
- телефон (обязательно);
- email (обязательно);
- комментарий (обязательно).

### Frontend

- Валидация до отправки;
- Состояния: **idle → loading → success | error**;
- Сообщения понятные пользователю;
- Honeypot (скрытое поле) — антиспам.

### API `POST /api/contact`

- Валидация Zod на сервере;
- Письмо **владельцу** (все поля);
- **Копия пользователю** («мы получили сообщение» + краткое резюме);
- Ответы: `200`, `400`, `405`, `429`, `500`;
- Rate limit по IP (базовый).

### Env (Vercel + локально)

```env
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USER=
SMTP_PASS=          # пароль приложения
MAIL_FROM=
MAIL_TO=
# опционально позже:
OPENAI_API_KEY=
```

Файл `.env` — только локально, в Git не коммитить.

## 8. Деплой (Vercel)

### Почему Vercel

- Статика + serverless API в одном проекте;
- Бесплатный Hobby: ~1M invocations/мес — с запасом для лендинга;
- Автодеплой из GitHub при push.

### Настройка (один раз)

1. Репозиторий на GitHub (публичный или private с доступом Vercel).
2. [vercel.com](https://vercel.com) → **Add New Project** → Import repo.
3. **Root Directory**: `/` (корень монорепо).
4. **Build Command**: `npm run build`
5. **Output Directory**: `apps/web/dist`
6. **Environment Variables** — переменные из §6.
7. Deploy → URL `*.vercel.app`.

### Поведение

- `push` в `main` → production;
- Pull Request → preview URL.

### Fallback

Если Vercel станет недоступен из РФ — VPS (Timeweb/Selectel) + Nginx + PM2 (описать в README одной строкой).
